# รายงานผลการทดสอบ

เอกสารนี้สรุปผลการทดสอบระบบหลังแก้ไขบั๊กสินค้าหายและเพิ่มระบบจัดการสินค้าครบวงจร (เพิ่ม แก้ไข ลบ) ทดสอบเมื่อวันที่ 22 กรกฎาคม 2569

## สรุปผลโดยรวม

ทุกรายการที่ทดสอบผ่านตามที่คาดหวัง ระบบพร้อมให้ทดสอบใช้งานจริง (User Acceptance Test) ตามเอกสาร [uat.md](./uat.md) ต่อไป

## วิธีการทดสอบ

ทดสอบ 2 ระดับควบคู่กัน

1. **ทดสอบผ่านเบราว์เซอร์จริง**: จำลองการใช้งานของผู้เยี่ยมชมและแอดมินผ่านเบราว์เซอร์อัตโนมัติ (ไม่ใช่แค่ดูโค้ดเฉยๆ) ครอบคลุมการดูสินค้า สมัครสมาชิก เข้าสู่ระบบ และจัดการสินค้า
2. **ทดสอบสิทธิ์การเข้าถึงข้อมูลโดยตรงกับฐานข้อมูล**: ยิงคำขอเข้าฐานข้อมูลโดยตรงโดยไม่ผ่านหน้าเว็บ เพื่อยืนยันว่ากฎความปลอดภัย (RLS) ทำงานถูกต้องจริง ไม่ได้ปลอดภัยแค่เพราะหน้าเว็บไม่มีปุ่มให้กด

## ผลการทดสอบรายรายการ

| รายการทดสอบ | ขั้นตอน | ผลที่คาดหวัง | ผลจริง |
| --- | --- | --- | --- |
| ผู้เยี่ยมชมเห็นสินค้าหน้าแรก | เปิดหน้าแรกโดยไม่เข้าสู่ระบบ | เห็นสินค้าทั้งหมดที่มีในฐานข้อมูล | ผ่าน เห็นสินค้าครบ 3 รายการ (ก่อนแก้ไขจะไม่เห็นสินค้าเลยเพราะบั๊กกฎความปลอดภัย) |
| สมัครสมาชิกใหม่ | กรอกชื่อ อีเมล รหัสผ่าน แล้วกดสมัครสมาชิก | สมัครสำเร็จ เข้าสู่ระบบได้ทันที | ผ่าน |
| เข้าสู่ระบบด้วยบัญชีแอดมิน | เข้าสู่ระบบด้วยบัญชีที่มีสิทธิ์ role เป็น admin แล้วเปิดหน้า /admin | เห็นแดชบอร์ดและระบบจัดการสินค้า | ผ่าน |
| แดชบอร์ดแสดงจำนวนสมาชิก | ดูตัวเลข "สมาชิกรวม" ในหน้าแอดมิน | แสดงจำนวนสมาชิกจริงในระบบ ไม่ใช่ 0 | ผ่าน (แสดง 4 คน ตรงกับจำนวนจริงในฐานข้อมูล ก่อนแก้ไขจะแสดง 0 เสมอ) |
| เพิ่มสินค้าใหม่ | กดปุ่มเพิ่มสินค้าใหม่ กรอกชื่อ ราคา รายละเอียด รูปภาพ แล้วกดบันทึก | สินค้าใหม่ปรากฏในรายการทันที | ผ่าน |
| แก้ไขสินค้าเดิม | เลือกสินค้าที่เพิ่งสร้าง แก้ชื่อ แล้วกดบันทึก | ชื่อสินค้าที่แก้ไขปรากฏในรายการ | ผ่าน |
| ลบสินค้า | เลือกสินค้าที่ทดสอบไว้ กดปุ่มลบ แล้วกดยืนยัน | สินค้าถูกลบออกจากระบบจริง | ผ่าน (ยืนยันด้วยการตรวจสอบข้อมูลในฐานข้อมูลโดยตรง) |
| ผู้เยี่ยมชม (ไม่เข้าสู่ระบบ) พยายามเพิ่มสินค้าโดยตรง | ส่งคำขอเพิ่มสินค้าเข้าฐานข้อมูลโดยตรง ไม่ผ่านหน้าเว็บ | ระบบปฏิเสธ ไม่ให้เพิ่มข้อมูลได้ | ผ่าน ระบบปฏิเสธคำขอ |
| ผู้เยี่ยมชม (ไม่เข้าสู่ระบบ) พยายามลบสินค้าโดยตรง | ส่งคำขอลบสินค้าเข้าฐานข้อมูลโดยตรง ไม่ผ่านหน้าเว็บ | ไม่มีสินค้าถูกลบจริง | ผ่าน ตรวจสอบแล้วสินค้ายังอยู่ครบ ไม่มีรายการใดถูกลบ |
| การสร้างโปรเจกต์ (build) ไม่มีข้อผิดพลาด | รันคำสั่งสร้างไฟล์เว็บไซต์สำหรับใช้งานจริง | สร้างสำเร็จไม่มี error | ผ่าน |

## ข้อสังเกตทางเทคนิคที่ควรทราบ

- เวลาส่งคำขอลบข้อมูลไปยังฐานข้อมูลโดยที่ไม่มีสิทธิ์ ระบบจะตอบกลับว่า "สำเร็จ" (สถานะ 204) แต่จริงๆ แล้วไม่มีข้อมูลถูกลบเลยแม้แต่แถวเดียว เพราะกฎความปลอดภัยกรองแถวออกไปตั้งแต่ต้นก่อนจะถึงขั้นตอนลบ พฤติกรรมนี้เป็นเรื่องปกติของระบบฐานข้อมูลที่ใช้ (ไม่ใช่บั๊ก) แต่ผู้ดูแลระบบควรทราบไว้เผื่อสับสนกับสถานะที่ได้รับ

## สิ่งที่ไม่ได้อยู่ในขอบเขตการทดสอบรอบนี้

การทดสอบรอบนี้เน้นเฉพาะเรื่องการแสดงสินค้าและการจัดการสินค้าในหน้าแอดมิน ไม่ได้ครอบคลุมเรื่องต่อไปนี้ เพราะยังไม่ได้พัฒนา (ดูรายละเอียดใน [analysis-and-design.md](./analysis-and-design.md))

- การยืนยันคำสั่งซื้อและชำระเงินจริง
- การตรวจสอบสลิปโอนเงินอัตโนมัติ
- หน้าประวัติคำสั่งซื้อของลูกค้า
- การแก้ไขข้อมูลโปรไฟล์ส่วนตัว (พบว่ามีบั๊กอยู่ระหว่างตรวจสอบ บันทึกไว้ในเอกสารการวิเคราะห์และออกแบบแล้ว)

# แผนภาพการใช้งานระบบ (Use Case และ Sequence Diagram)

เอกสารนี้ใช้ภาพช่วยอธิบายว่าใครทำอะไรได้บ้างในระบบ และแต่ละการทำงานมีขั้นตอนอย่างไร อ่านควบคู่กับ [how-it-works.md](./how-it-works.md) จะเห็นภาพชัดขึ้น

## แผนภาพการใช้งานระบบ (Use Case Diagram)

แผนภาพนี้แสดงว่าผู้ใช้แต่ละแบบทำอะไรกับระบบได้บ้าง

```mermaid
flowchart LR
    subgraph ผู้ใช้งาน
        guest((ผู้เยี่ยมชม))
        customer((ลูกค้าสมาชิก))
        admin((แอดมิน))
    end

    subgraph การทำงานของระบบ
        uc1[ดูรายการสินค้า]
        uc2[ใส่สินค้าลงตะกร้า]
        uc3[สมัครสมาชิก / เข้าสู่ระบบ]
        uc4[แก้ไขข้อมูลส่วนตัว]
        uc5[ดูประวัติคำสั่งซื้อ]
        uc6[เพิ่มสินค้าใหม่]
        uc7[แก้ไขข้อมูลสินค้า]
        uc8[ลบสินค้า]
        uc9[ดูแดชบอร์ดสรุปยอดขาย]
    end

    guest --> uc1
    guest --> uc2
    guest --> uc3

    customer --> uc1
    customer --> uc2
    customer --> uc4
    customer --> uc5

    admin --> uc1
    admin --> uc6
    admin --> uc7
    admin --> uc8
    admin --> uc9
```

หมายเหตุ: "ดูประวัติคำสั่งซื้อ" ในแผนภาพนี้หมายถึงหน้าที่มีอยู่ในเว็บไซต์ แต่ปัจจุบันยังเป็นหน้าว่างเปล่าเพราะระบบสั่งซื้อจริงยังไม่ถูกพัฒนา (ดู [analysis-and-design.md](./analysis-and-design.md))

## Sequence Diagram: ผู้เยี่ยมชมดูสินค้าหน้าแรก

```mermaid
sequenceDiagram
    participant ผู้ใช้ as ผู้เยี่ยมชม
    participant เว็บ as หน้าเว็บ (React)
    participant DB as ฐานข้อมูล Supabase

    ผู้ใช้->>เว็บ: เปิดหน้าแรกของเว็บไซต์
    เว็บ->>DB: ขอรายการสินค้าทั้งหมด
    DB-->>เว็บ: ส่งรายการสินค้ากลับมา
    เว็บ-->>ผู้ใช้: แสดงการ์ดสินค้าพร้อมราคาและรูปภาพ
```

## Sequence Diagram: สมัครสมาชิกและเข้าสู่ระบบ

```mermaid
sequenceDiagram
    participant ผู้ใช้ as ผู้ใช้งาน
    participant เว็บ as หน้าเว็บ (React)
    participant Auth as ระบบสมาชิก Supabase
    participant DB as ฐานข้อมูล

    ผู้ใช้->>เว็บ: กรอกชื่อ อีเมล รหัสผ่าน แล้วกดสมัครสมาชิก
    เว็บ->>Auth: ขอสร้างบัญชีใหม่
    Auth-->>DB: สร้างแถวโปรไฟล์เริ่มต้นให้อัตโนมัติ (สิทธิ์เป็นลูกค้าทั่วไป)
    Auth-->>เว็บ: แจ้งผลว่าสมัครสำเร็จ
    เว็บ-->>ผู้ใช้: แสดงข้อความสมัครสมาชิกสำเร็จ

    ผู้ใช้->>เว็บ: กรอกอีเมลและรหัสผ่านเพื่อเข้าสู่ระบบ
    เว็บ->>Auth: ขอเข้าสู่ระบบ
    Auth-->>เว็บ: ส่งข้อมูลการเข้าสู่ระบบ (session) กลับมา
    เว็บ-->>ผู้ใช้: เข้าสู่ระบบสำเร็จ แสดงชื่อผู้ใช้มุมขวาบน
```

## Sequence Diagram: แอดมินเพิ่มสินค้าใหม่

```mermaid
sequenceDiagram
    participant แอดมิน as แอดมิน
    participant เว็บ as หน้าแอดมิน (React)
    participant DB as ฐานข้อมูล Supabase

    แอดมิน->>เว็บ: เข้าหน้า /admin
    เว็บ->>DB: ตรวจสอบสิทธิ์ผู้ใช้ (role เป็น admin หรือไม่)
    DB-->>เว็บ: ยืนยันว่าเป็นแอดมิน
    เว็บ-->>แอดมิน: แสดงหน้าแดชบอร์ดและฟอร์มจัดการสินค้า

    แอดมิน->>เว็บ: กดปุ่ม "เพิ่มสินค้าใหม่" กรอกข้อมูล แล้วกดบันทึก
    เว็บ->>DB: ส่งคำขอเพิ่มสินค้าใหม่
    DB-->>เว็บ: ยืนยันว่าเพิ่มสำเร็จ
    เว็บ->>DB: ขอรายการสินค้าล่าสุด
    DB-->>เว็บ: ส่งรายการสินค้าที่อัปเดตแล้ว
    เว็บ-->>แอดมิน: แสดงข้อความ "เพิ่มสินค้าใหม่เรียบร้อย" และรายการสินค้าล่าสุด
```

## Sequence Diagram: แอดมินลบสินค้า

```mermaid
sequenceDiagram
    participant แอดมิน as แอดมิน
    participant เว็บ as หน้าแอดมิน (React)
    participant DB as ฐานข้อมูล Supabase

    แอดมิน->>เว็บ: เลือกสินค้าที่ต้องการลบ แล้วกด "ลบสินค้านี้"
    เว็บ-->>แอดมิน: แสดงกล่องยืนยัน "ยืนยันลบสินค้านี้ออกจากระบบ"
    แอดมิน->>เว็บ: กดยืนยัน
    เว็บ->>DB: ส่งคำขอลบสินค้าตามรหัสสินค้า
    DB-->>เว็บ: ยืนยันว่าลบสำเร็จ
    เว็บ-->>แอดมิน: แสดงข้อความ "ลบสินค้าเรียบร้อย" และตัดสินค้าออกจากรายการ
```

## Sequence Diagram: ผู้ใช้ทั่วไปพยายามเข้าหน้าแอดมิน (ถูกปฏิเสธสิทธิ์)

```mermaid
sequenceDiagram
    participant ลูกค้า as ลูกค้าทั่วไป
    participant เว็บ as หน้าแอดมิน (React)
    participant DB as ฐานข้อมูล Supabase

    ลูกค้า->>เว็บ: พิมพ์ที่อยู่เว็บ /admin เอง
    เว็บ->>DB: ตรวจสอบสิทธิ์ผู้ใช้
    DB-->>เว็บ: role เป็น customer ไม่ใช่ admin
    เว็บ-->>ลูกค้า: แสดงข้อความ "ไม่มีสิทธิ์เข้าถึง" ไม่แสดงข้อมูลใดๆ เพิ่มเติม
```

# แผนภาพการใช้งานระบบ (Use Case และ Sequence Diagram)

เอกสารนี้ใช้ภาพช่วยอธิบายว่าใครทำอะไรได้บ้างในระบบ และแต่ละการทำงานมีขั้นตอนอย่างไร อ่านควบคู่กับ [how-it-works.md](./how-it-works.md) และ [personas.md](./personas.md) จะเห็นภาพชัดขึ้น

## แผนภาพการใช้งานระบบ (Use Case Diagram)

แผนภาพนี้แสดงว่าผู้ใช้แต่ละแบบทำอะไรกับระบบได้บ้าง

```mermaid
flowchart LR

    subgraph USERS["ผู้ใช้งาน"]
        guest((ผู้เยี่ยมชม))
        customer((ลูกค้าสมาชิก))
        admin((แอดมิน))
    end

    subgraph SYSTEM["การทำงานของระบบ"]
        uc1["ดูรายการสินค้า"]
        uc10["ค้นหา/กรองสินค้าตามหมวดหมู่"]
        uc11["ดูรายละเอียดสินค้า"]
        uc2["ใส่สินค้าลงตะกร้า"]
        uc3["สมัครสมาชิก / เข้าสู่ระบบ"]
        uc4["แก้ไขข้อมูลส่วนตัว"]
        uc12["ยืนยันคำสั่งซื้อ"]
        uc13["ดูใบเสร็จ/ติดตามสถานะคำสั่งซื้อ"]
        uc5["ดูประวัติคำสั่งซื้อ"]
        uc6["เพิ่มสินค้าใหม่"]
        uc7["แก้ไขข้อมูลสินค้า"]
        uc8["ลบสินค้า"]
        uc14["จัดการคำสั่งซื้อ/เปลี่ยนสถานะ"]
        uc15["จัดการสต็อกสินค้า"]
        uc9["ดูแดชบอร์ดสรุปยอดขาย"]
    end

    guest --> uc1
    guest --> uc10
    guest --> uc11
    guest --> uc2
    guest --> uc3

    customer --> uc1
    customer --> uc10
    customer --> uc11
    customer --> uc2
    customer --> uc4
    customer --> uc12
    customer --> uc13
    customer --> uc5

    admin --> uc1
    admin --> uc6
    admin --> uc7
    admin --> uc8
    admin --> uc14
    admin --> uc15
    admin --> uc9
```

**หมายเหตุ:** `uc12` (ยืนยันคำสั่งซื้อ) และ `uc13`/`uc5` (ดูสถานะ/ประวัติคำสั่งซื้อ) ต้องเข้าสู่ระบบก่อน ผู้เยี่ยมชมสามารถใส่สินค้าในตะกร้าและสมัครสมาชิกได้เท่านั้น `uc15` (จัดการสต็อก) รวมถึงการดูวิดเจ็ตสินค้าใกล้หมดและแก้ไขจำนวนสต็อกในฟอร์มสินค้า

## Sequence Diagram: ผู้เยี่ยมชมดูสินค้าหน้าแรก

```mermaid
sequenceDiagram
    participant ผู้ใช้ as ผู้เยี่ยมชม
    participant เว็บ as หน้าเว็บ (React)
    participant DB as ฐานข้อมูล Supabase

    ผู้ใช้->>เว็บ: เปิดหน้าแรกของเว็บไซต์
    เว็บ->>DB: ขอรายการสินค้าทั้งหมด
    DB-->>เว็บ: ส่งรายการสินค้ากลับมา
    เว็บ-->>ผู้ใช้: แสดงการ์ดสินค้าพร้อมราคาและรูปภาพ
```

## Sequence Diagram: ค้นหาและกรองสินค้าตามหมวดหมู่

```mermaid
sequenceDiagram
    participant ผู้ใช้ as ผู้เยี่ยมชม/ลูกค้า
    participant เว็บ as หน้าแรก (React)

    ผู้ใช้->>เว็บ: พิมพ์คำค้นหา หรือเลือกหมวดหมู่จาก dropdown
    เว็บ->>เว็บ: กรองรายการสินค้าที่โหลดไว้แล้วในเครื่อง (ไม่ยิงคำขอใหม่ไปฐานข้อมูล)
    เว็บ-->>ผู้ใช้: แสดงเฉพาะสินค้าที่ตรงกับคำค้นหา/หมวดหมู่ (หรือข้อความ "ไม่พบสินค้าที่ค้นหา")
```

## Sequence Diagram: ดูรายละเอียดสินค้า

```mermaid
sequenceDiagram
    participant ผู้ใช้ as ผู้เยี่ยมชม/ลูกค้า
    participant เว็บ as หน้ารายละเอียดสินค้า (React)
    participant DB as ฐานข้อมูล Supabase

    ผู้ใช้->>เว็บ: คลิกรูปหรือชื่อสินค้าจากหน้าแรก
    เว็บ->>DB: ขอข้อมูลสินค้าตามรหัสสินค้า (id)
    DB-->>เว็บ: ส่งข้อมูลสินค้ากลับมา (หรือไม่พบ)
    เว็บ-->>ผู้ใช้: แสดงรูปใหญ่ ชื่อ หมวดหมู่ คำอธิบายเต็ม ราคา และปุ่มหยิบใส่ตะกร้า (หรือข้อความ "ไม่พบสินค้านี้")
```

## Sequence Diagram: สมัครสมาชิกและเข้าสู่ระบบ

```mermaid
sequenceDiagram
    participant ผู้ใช้ as ผู้ใช้งาน
    participant เว็บ as หน้าเว็บ (React)
    participant Auth as ระบบสมาชิก Supabase
    participant DB as ฐานข้อมูล

    ผู้ใช้->>เว็บ: กรอกชื่อ อีเมล รหัสผ่าน แล้วกดสมัครสมาชิก
    เว็บ->>Auth: ขอสร้างบัญชีใหม่
    Auth-->>DB: สร้างแถวโปรไฟล์เริ่มต้นให้อัตโนมัติ (สิทธิ์เป็นลูกค้าทั่วไป)
    Auth-->>เว็บ: แจ้งผลว่าสมัครสำเร็จ
    เว็บ-->>ผู้ใช้: แสดงข้อความสมัครสมาชิกสำเร็จ

    ผู้ใช้->>เว็บ: กรอกอีเมลและรหัสผ่านเพื่อเข้าสู่ระบบ
    เว็บ->>Auth: ขอเข้าสู่ระบบ
    Auth-->>เว็บ: ส่งข้อมูลการเข้าสู่ระบบ (session) กลับมา
    เว็บ-->>ผู้ใช้: เข้าสู่ระบบสำเร็จ แสดงชื่อผู้ใช้มุมขวาบน
```

## Sequence Diagram: ลูกค้ายืนยันคำสั่งซื้อ

```mermaid
sequenceDiagram
    participant ลูกค้า as ลูกค้าสมาชิก
    participant เว็บ as หน้าตะกร้า (React)
    participant DB as ฐานข้อมูล Supabase

    ลูกค้า->>เว็บ: กรอกที่อยู่จัดส่ง (ชื่อ เบอร์โทร ที่อยู่) และกดปุ่ม "ยืนยันคำสั่งซื้อ"
    เว็บ->>เว็บ: ตรวจสอบว่ากรอกที่อยู่จัดส่งครบหรือไม่
    alt กรอกไม่ครบ
        เว็บ-->>ลูกค้า: แจ้งช่องที่ขาด (ชื่อ/เบอร์/ที่อยู่) ไม่ดำเนินการต่อ
    else กรอกครบแล้ว
        เว็บ->>เว็บ: ตรวจสอบว่าเข้าสู่ระบบอยู่หรือไม่
        alt ยังไม่เข้าสู่ระบบ
            เว็บ-->>ลูกค้า: แจ้ง "กรุณาเข้าสู่ระบบก่อนสั่งซื้อ" (ไม่สร้างคำสั่งซื้อ)
        else เข้าสู่ระบบแล้ว
            เว็บ->>DB: ขอตรวจสอบสต็อกสินค้าทุกรายการในตะกร้า
            DB-->>เว็บ: ส่งจำนวนสต็อกปัจจุบันกลับมา
            alt สต็อกไม่พอ
                เว็บ-->>ลูกค้า: แจ้งชื่อสินค้าที่สต็อกไม่พอและจำนวนที่เหลือ (ไม่สร้างคำสั่งซื้อ)
            else สต็อกพอ
                เว็บ->>DB: สร้างคำสั่งซื้อใหม่ในตาราง orders พร้อม shipping_name/phone/address (status = pending)
                DB-->>เว็บ: ส่งรหัสคำสั่งซื้อ (order id) กลับมา
                เว็บ->>DB: บันทึกรายการสินค้าทั้งหมดลงตาราง order_items
                DB-->>เว็บ: ยืนยันบันทึกสำเร็จ
                เว็บ->>DB: หักสต็อกสินค้าแต่ละรายการตามจำนวนที่สั่ง
                DB-->>เว็บ: ยืนยันอัปเดตสต็อกสำเร็จ
                เว็บ->>เว็บ: ล้างตะกร้าสินค้า
                เว็บ-->>ลูกค้า: พาไปหน้าใบเสร็จ /order/:id
            end
        end
    end
```

## Sequence Diagram: ลูกค้าดูใบเสร็จและติดตามสถานะคำสั่งซื้อ

```mermaid
sequenceDiagram
    participant ลูกค้า as ลูกค้าสมาชิก
    participant เว็บ as หน้าใบเสร็จ/ประวัติ (React)
    participant DB as ฐานข้อมูล Supabase

    ลูกค้า->>เว็บ: เปิดหน้า "ประวัติการสั่งซื้อ" หรือกลับมาที่ /order/:id
    เว็บ->>DB: ขอคำสั่งซื้อของผู้ใช้คนนี้ (กรองด้วย user_id ผ่าน RLS)
    DB-->>เว็บ: ส่งรายการคำสั่งซื้อพร้อมสถานะล่าสุด
    เว็บ-->>ลูกค้า: แสดงวันที่ ยอดรวม และ badge สถานะ (รอดำเนินการ/ชำระเงินแล้ว/จัดส่งแล้ว/ยกเลิก)
```

## Sequence Diagram: แอดมินเพิ่มสินค้าใหม่

```mermaid
sequenceDiagram
    participant แอดมิน as แอดมิน
    participant เว็บ as หน้าแอดมิน (React)
    participant DB as ฐานข้อมูล Supabase

    แอดมิน->>เว็บ: เข้าหน้า /admin
    เว็บ->>DB: ตรวจสอบสิทธิ์ผู้ใช้ (role เป็น admin หรือไม่)
    DB-->>เว็บ: ยืนยันว่าเป็นแอดมิน
    เว็บ-->>แอดมิน: แสดงหน้าแดชบอร์ดและฟอร์มจัดการสินค้า

    แอดมิน->>เว็บ: กดปุ่ม "เพิ่มสินค้าใหม่" กรอกข้อมูล+หมวดหมู่+จำนวนสต็อก แล้วกดบันทึก
    เว็บ->>DB: ส่งคำขอเพิ่มสินค้าใหม่
    DB-->>เว็บ: ยืนยันว่าเพิ่มสำเร็จ
    เว็บ->>DB: ขอรายการสินค้าล่าสุด
    DB-->>เว็บ: ส่งรายการสินค้าที่อัปเดตแล้ว
    เว็บ-->>แอดมิน: แสดงข้อความ "เพิ่มสินค้าใหม่เรียบร้อย" และรายการสินค้าล่าสุด
```

## Sequence Diagram: แอดมินลบสินค้า

```mermaid
sequenceDiagram
    participant แอดมิน as แอดมิน
    participant เว็บ as หน้าแอดมิน (React)
    participant DB as ฐานข้อมูล Supabase

    แอดมิน->>เว็บ: เลือกสินค้าที่ต้องการลบ แล้วกด "ลบสินค้านี้"
    เว็บ-->>แอดมิน: แสดงกล่องยืนยัน "ยืนยันลบสินค้านี้ออกจากระบบ"
    แอดมิน->>เว็บ: กดยืนยัน
    เว็บ->>DB: ส่งคำขอลบสินค้าตามรหัสสินค้า
    DB-->>เว็บ: ยืนยันว่าลบสำเร็จ
    เว็บ-->>แอดมิน: แสดงข้อความ "ลบสินค้าเรียบร้อย" และตัดสินค้าออกจากรายการ
```

## Sequence Diagram: แอดมินจัดการคำสั่งซื้อ (เปลี่ยนสถานะ)

```mermaid
sequenceDiagram
    participant แอดมิน as แอดมิน
    participant เว็บ as หน้าแอดมิน (React)
    participant DB as ฐานข้อมูล Supabase
    participant ลูกค้า as ลูกค้าเจ้าของคำสั่งซื้อ

    แอดมิน->>เว็บ: เปิด section "จัดการคำสั่งซื้อ" ในหน้า /admin
    เว็บ->>DB: ขอคำสั่งซื้อทั้งหมด พร้อมชื่อ/อีเมลผู้ซื้อ
    DB-->>เว็บ: ส่งรายการคำสั่งซื้อทั้งหมดกลับมา
    เว็บ-->>แอดมิน: แสดงตารางคำสั่งซื้อพร้อม dropdown เปลี่ยนสถานะ

    แอดมิน->>เว็บ: เลือกสถานะใหม่ (เช่น "ชำระเงินแล้ว") จาก dropdown
    เว็บ->>DB: อัปเดตสถานะคำสั่งซื้อ (ตรวจสิทธิ์ผ่าน policy orders_update_admin)
    DB-->>เว็บ: ยืนยันอัปเดตสำเร็จ
    เว็บ-->>แอดมิน: อัปเดตตารางทันทีโดยไม่ต้อง refresh
    ลูกค้า->>เว็บ: เปิดหน้าประวัติ/ใบเสร็จของตัวเองอีกครั้ง
    เว็บ-->>ลูกค้า: เห็นสถานะที่เปลี่ยนแล้ว
```

## Sequence Diagram: แอดมินจัดการสต็อกสินค้า

```mermaid
sequenceDiagram
    participant แอดมิน as แอดมิน
    participant เว็บ as หน้าแอดมิน (React)
    participant DB as ฐานข้อมูล Supabase

    แอดมิน->>เว็บ: เปิดแท็บ "ภาพรวม" ดูวิดเจ็ตสินค้าใกล้หมด
    เว็บ->>DB: ขอรายการสินค้าที่มี stock <= 3
    DB-->>เว็บ: ส่งรายการสินค้าสต็อกต่ำกลับมา
    เว็บ-->>แอดมิน: แสดงรายชื่อสินค้าพร้อมจำนวนสต็อกที่เหลือ

    แอดมิน->>เว็บ: ไปแท็บ "สินค้า" เลือกสินค้าที่ต้องการเติมสต็อก
    เว็บ-->>แอดมิน: แสดงฟอร์มแก้ไขพร้อมจำนวนสต็อกปัจจุบัน
    แอดมิน->>เว็บ: แก้ไขจำนวนสต็อก แล้วกดบันทึก
    เว็บ->>DB: อัปเดต stock ของสินค้านั้น
    DB-->>เว็บ: ยืนยันอัปเดตสำเร็จ
    เว็บ-->>แอดมิน: แสดงข้อความ "บันทึกเรียบร้อย" สต็อกถูกอัปเดตทันที
```

## Sequence Diagram: ผู้ใช้ทั่วไปพยายามเข้าหน้าแอดมิน (ถูกปฏิเสธสิทธิ์)

sequenceDiagram

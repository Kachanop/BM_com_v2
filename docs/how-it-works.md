# ระบบทำงานอย่างไร

เอกสารนี้อธิบายการทำงานของเว็บไซต์แบบภาษาคน ไม่เน้นศัพท์เทคนิค เหมาะสำหรับคนที่อยากเข้าใจภาพรวมก่อนลงลึกเรื่องโค้ด

## ผู้ใช้งานระบบมีกี่แบบ

1. **ผู้เยี่ยมชมทั่วไป (ยังไม่ได้สมัครสมาชิก)** ดูสินค้า ค้นหา กรองหมวดหมู่ ดูหน้ารายละเอียดสินค้า และใส่ตะกร้าได้ แต่ต้องเข้าสู่ระบบก่อนถึงจะยืนยันสั่งซื้อหรือดูประวัติคำสั่งซื้อได้
2. **ลูกค้าที่เป็นสมาชิก** สมัครและเข้าสู่ระบบได้ แก้ไขข้อมูลส่วนตัวได้ ดูตะกร้าและประวัติคำสั่งซื้อของตัวเองได้
3. **แอดมิน** เป็นสมาชิกพิเศษที่มีสิทธิ์เพิ่มเติม สามารถเข้าหน้าจัดการระบบหลังบ้าน เพิ่ม แก้ไข และลบสินค้าได้ พร้อมดูสถิติยอดขาย

## ลูกค้าดูและเลือกซื้อสินค้าอย่างไร

1. เปิดหน้าแรกของเว็บไซต์ ระบบจะไปดึงรายการสินค้าทั้งหมดจากฐานข้อมูลมาแสดง
2. พิมพ์คำค้นหาในช่องค้นหา หรือกดปุ่มกรองหมวดหมู่ (pill button) เพื่อกรองสินค้าที่เห็น — เป็นการกรองจากรายการสินค้าที่โหลดมาแล้วในเครื่อง ไม่ได้ค้นหาใหม่จากฐานข้อมูลทุกครั้งที่พิมพ์
3. คลิกที่รูปหรือชื่อสินค้าเพื่อเปิดหน้ารายละเอียดสินค้า เห็นคำอธิบายเต็มและหมวดหมู่ พร้อมกดหยิบใส่ตะกร้าจากหน้านั้นได้เช่นกัน
4. กดปุ่ม "หยิบใส่ตะกร้า" สินค้าจะถูกเก็บไว้ในเครื่องของผู้ใช้เอง (ไม่ได้ส่งขึ้นฐานข้อมูลจนกว่าจะสั่งซื้อจริง) ดังนั้นถ้าปิดเบราว์เซอร์แล้วเปิดใหม่ สินค้าที่ใส่ตะกร้าไว้จะยังอยู่
5. ไปหน้าตะกร้าเพื่อดูรายการที่เลือกไว้ ปรับจำนวน หรือเอาออกได้

## ลูกค้ายืนยันสั่งซื้อและดูสถานะอย่างไร

1. ที่หน้าตะกร้า กรอกที่อยู่จัดส่ง (ชื่อ-นามสกุลผู้รับ เบอร์โทร และที่อยู่เต็ม) แล้วกดปุ่ม "ยืนยันคำสั่งซื้อ" — ถ้ายังไม่ได้เข้าสู่ระบบ ระบบจะแจ้งให้เข้าสู่ระบบก่อน ยังไม่สร้างคำสั่งซื้อใดๆ
2. ระบบตรวจสอบสต็อกสินค้าในฐานข้อมูลอีกครั้งก่อนสร้างคำสั่งซื้อ ถ้าสินค้าใดหมดหรือเหลือน้อยกว่าที่ต้องการจะแจ้งเตือนทันที
3. ถ้าสต็อกเพียงพอ ระบบจะบันทึกคำสั่งซื้อพร้อมที่อยู่จัดส่งลงฐานข้อมูล (สถานะเริ่มต้นคือ "รอดำเนินการ") และหักสต็อกสินค้าออกอัตโนมัติ แล้วล้างตะกร้าให้อัตโนมัติ
4. ระบบพาไปหน้าใบเสร็จ แสดงเลขคำสั่งซื้อ รายการสินค้า ที่อยู่จัดส่ง ยอดรวม และกล่อง QR พร้อมเพย์ตัวอย่าง (ยังไม่ใช่เกตเวย์ชำระเงินจริง) พร้อมปุ่มพิมพ์ใบเสร็จ
5. ลูกค้ากลับมาดูสถานะคำสั่งซื้อของตัวเองได้ทุกเมื่อที่หน้า "ประวัติการสั่งซื้อ" สถานะจะเปลี่ยนจาก "รอดำเนินการ" เป็น "ชำระเงินแล้ว" หรือ "จัดส่งแล้ว" ก็ต่อเมื่อแอดมินเข้าไปเปลี่ยนให้ด้วยมือ (ยังไม่มีระบบตรวจสลิปโอนเงินอัตโนมัติ)

## การสมัครสมาชิกและเข้าสู่ระบบ

1. ผู้ใช้กดปุ่ม "เข้าสู่ระบบ" มุมขวาบน แล้วเลือกแท็บ "สมัครสมาชิกใหม่"
2. กรอกชื่อ อีเมล และรหัสผ่าน ระบบจะสร้างบัญชีผ่าน Supabase Auth ให้อัตโนมัติ
3. เมื่อสมัครสำเร็จ ระบบจะสร้างข้อมูลโปรไฟล์เริ่มต้นของผู้ใช้คนนั้นในฐานข้อมูลให้เองเบื้องหลัง (ผ่านกลไกอัตโนมัติที่ตั้งไว้ในฐานข้อมูล) โดยตั้งสิทธิ์เริ่มต้นเป็น "ลูกค้าทั่วไป"
4. ผู้ใช้เข้าสู่ระบบด้วยอีเมลและรหัสผ่านที่ตั้งไว้

## แอดมินตรวจสอบสิทธิ์อย่างไรก่อนเข้าหน้าหลังบ้าน

เว็บไซต์จะตรวจสิทธิ์ 2 ชั้นก่อนให้เข้าหน้าแอดมิน

1. **ต้องเข้าสู่ระบบก่อน** ถ้ายังไม่เข้าสู่ระบบ จะเห็นข้อความ "สงวนสิทธิ์เฉพาะผู้ดูแลระบบ"
2. **ต้องมีสิทธิ์เป็นแอดมินในฐานข้อมูล** ระบบจะไปตรวจข้อมูลโปรไฟล์ของผู้ใช้คนนั้นว่ามีคอลัมน์ "role" เป็นค่า "admin" หรือไม่ ถ้าไม่ใช่ จะเห็นข้อความ "ไม่มีสิทธิ์เข้าถึง" แม้จะเข้าสู่ระบบสำเร็จแล้วก็ตาม

การกำหนดว่าใครเป็นแอดมินทำผ่านฐานข้อมูลโดยตรง (แก้ค่า role ในตาราง profiles) ไม่มีปุ่มในหน้าเว็บให้ผู้ใช้ตั้งตัวเองเป็นแอดมิน เพื่อความปลอดภัย

## แอดมินจัดการสินค้าอย่างไร

หน้าแอดมินมีระบบจัดการสินค้าครบ 4 การทำงาน

1. **ดูรายการสินค้าทั้งหมด** แสดงเป็นรายการในแท็บ "สินค้า"
2. **เพิ่มสินค้าใหม่** กดปุ่ม "เพิ่มสินค้าใหม่" แล้วกรอกชื่อ ราคา รายละเอียด ลิงก์รูปภาพ หมวดหมู่สินค้า และจำนวนสต็อก
3. **แก้ไขสินค้าเดิม** กดเลือกสินค้าจากรายการ แล้วแก้ข้อมูลในฟอร์ม (รวมถึงหมวดหมู่และสต็อก) กดบันทึก
4. **ลบสินค้า** เลือกสินค้าที่ต้องการลบ แล้วกดปุ่ม "ลบสินค้านี้" ระบบจะถามยืนยันก่อนลบจริงทุกครั้ง กันการลบพลาด

ทุกการเปลี่ยนแปลงจะถูกบันทึกลงฐานข้อมูลทันที และหน้าแรกของลูกค้าจะเห็นการเปลี่ยนแปลงนี้ในครั้งถัดไปที่โหลดหน้า

## แอดมินดูและจัดการสต็อกสินค้าอย่างไร

แดชบอร์ดแอดมินมีวิดเจ็ต "สินค้าใกล้หมด" แสดงรายการสินค้าที่สต็อกเหลือ 3 ชิ้นหรือน้อยกว่า พร้อมจำนวนคงเหลือ ช่วยให้แอดมินเติมสต็อกได้ทันก่อนสินค้าหมด แอดมินสามารถแก้ไขจำนวนสต็อกได้ในฟอร์มแก้ไขสินค้า เมื่อลูกค้าสั่งซื้อสำเร็จ ระบบจะหักสต็อกออกอัตโนมัติตามจำนวนที่สั่ง สินค้าที่สต็อก = 0 จะแสดงป้าย "สินค้าหมด" บนหน้าแรกและปุ่มหยิบใส่ตะกร้าจะถูกปิดใช้งานอัตโนมัติ

## แอดมินจัดการคำสั่งซื้ออย่างไร

หน้าแอดมินมีตาราง "จัดการคำสั่งซื้อ" แสดงคำสั่งซื้อทุกรายการในระบบ พร้อมวันที่ ชื่อ/อีเมลผู้ซื้อ ยอดรวม และที่อยู่จัดส่ง (ชื่อผู้รับ เบอร์โทร และที่อยู่) แอดมินเปลี่ยนสถานะได้ทันทีผ่าน dropdown ในตาราง (รอดำเนินการ / ชำระเงินแล้ว / จัดส่งแล้ว / ยกเลิก) โดยทั่วไปแอดมินจะตรวจสอบยอดโอนเงินจริงจากธนาคารก่อน แล้วค่อยเปลี่ยนสถานะเป็น "ชำระเงินแล้ว" ด้วยมือ เพราะระบบยังไม่มีการตรวจสลิปอัตโนมัติ ลูกค้าจะเห็นสถานะที่เปลี่ยนแล้วทันทีที่หน้าประวัติคำสั่งซื้อหรือใบเสร็จของตัวเอง

## แดชบอร์ดสรุปยอดขายในหน้าแอดมิน

หน้าแอดมินจะแสดงตัวเลขสรุป 4 อย่างในแท็บ "ภาพรวม"

- ยอดขายรวม (รวมยอดจากคำสั่งซื้อทั้งหมดในระบบ)
- จำนวนคำสั่งซื้อ
- จำนวนสมาชิกทั้งหมด
- จำนวนสินค้าในระบบ

นอกจากนี้ยังมีวิดเจ็ต "สินค้าใกล้หมด" แสดงรายการสินค้าที่เหลือสต็อก 3 ชิ้นหรือน้อยกว่า ตัวเลขทั้งหมดจะอัปเดตตามข้อมูลจริงในฐานข้อมูล

# Persona Design

เอกสารนี้อธิบาย Persona (ตัวแทนกลุ่มผู้ใช้งานจริง) ของระบบ เพื่อใช้เป็นหลักในการออกแบบหน้าจอและฟีเจอร์ให้ตรงกับความต้องการของผู้ใช้แต่ละกลุ่ม สร้างขึ้นจาก 3 บทบาทหลักที่ระบบรองรับ (ดู [use-case-and-sequence.md](./use-case-and-sequence.md))

## Persona 1: ผู้เยี่ยมชม (Guest) — "น้องเฟิร์ส นักศึกษาปี 2"

| หัวข้อ | รายละเอียด |
| --- | --- |
| อายุ / อาชีพ | 19 ปี นักศึกษาคณะวิศวกรรมคอมพิวเตอร์ |
| ความรู้ด้านเทคนิค | ใช้คอมพิวเตอร์เป็นประจำ แต่ไม่รู้จักสเปคฮาร์ดแวร์ลึกๆ |
| เป้าหมาย | อยากได้คอมสำหรับเรียนเขียนโปรแกรมและเล่นเกมเบาๆ ในงบไม่เกิน 25,000 บาท โดยไม่ต้องจัดสเปคเอง |
| Pain Point | เคยลองไปซื้อคอมแยกชิ้นแต่งงเรื่องสเปคที่เข้ากันได้ กลัวซื้อของไม่ตรงกับที่ใช้งานจริง ไม่อยากเสียเวลาคุยกับพนักงานขายที่ร้าน |
| พฤติกรรมการใช้เว็บ | เข้าเว็บผ่านมือถือระหว่างพักเบรกเรียน ดูราคาก่อนตัดสินใจ ชอบเปรียบเทียบสินค้าหลายรุ่นก่อนตัดสินใจซื้อ |
| สถานการณ์ใช้งาน (Scenario) | เปิดหน้าแรก พิมพ์คำว่า "เกมมิ่ง" ในช่องค้นหา กรองหมวดหมู่เป็น "เกมมิ่ง" ดูรายละเอียดสินค้าที่สนใจ 2-3 รุ่น เปรียบเทียบราคาและสเปค แล้วหยิบใส่ตะกร้าไว้ก่อนโดยยังไม่สมัครสมาชิก |
| สิ่งที่ระบบต้องรองรับ | ดูสินค้า/ค้นหา/กรองหมวดหมู่/ดูรายละเอียดสินค้า/ใส่ตะกร้าได้โดยไม่ต้องสมัครสมาชิกก่อน |

## Persona 2: ลูกค้าสมาชิก (Customer) — "พี่มายด์ พนักงานออฟฟิศ"

| หัวข้อ | รายละเอียด |
| --- | --- |
| อายุ / อาชีพ | 29 ปี พนักงานบัญชีบริษัทเอกชน |
| ความรู้ด้านเทคนิค | ใช้คอมทำงานทุกวัน แต่ไม่ถนัดเรื่องเทคนิคคอมพิวเตอร์ |
| เป้าหมาย | อยากได้คอมตั้งโต๊ะสำหรับทำงานเอกสาร/ประชุมออนไลน์ ที่มั่นใจได้ว่าใช้งานได้จริงทันทีที่ได้รับของ ไม่ต้องประกอบเอง |
| Pain Point | กลัวโอนเงินแล้วไม่ได้ของ ไม่มีเวลาติดตามสถานะคำสั่งซื้อบ่อยๆ อยากรู้ว่าออเดอร์ถึงไหนแล้วโดยไม่ต้องโทรถามร้าน |
| พฤติกรรมการใช้เว็บ | สั่งซื้อผ่านคอมที่ทำงาน เช็คอีเมล/เว็บไซต์ช่วงเย็นหลังเลิกงาน ให้ความสำคัญกับใบเสร็จที่เอาไปเบิกบริษัทได้ |
| สถานการณ์ใช้งาน (Scenario) | สมัครสมาชิกด้วยอีเมลบริษัท เลือกสินค้าใส่ตะกร้า กดยืนยันคำสั่งซื้อ ได้ใบเสร็จพร้อม QR พร้อมเพย์ตัวอย่าง โอนเงินจริงผ่านแอปธนาคาร แล้วกลับมาเช็คสถานะที่หน้า "ประวัติการสั่งซื้อ" เป็นระยะจนกว่าจะขึ้นสถานะ "จัดส่งแล้ว" |
| สิ่งที่ระบบต้องรองรับ | สมัคร/เข้าสู่ระบบ, ยืนยันคำสั่งซื้อ, ดูใบเสร็จ, ติดตามสถานะคำสั่งซื้อของตัวเองได้ตลอดเวลา |

## Persona 3: แอดมิน (Admin) — "พี่เอก เจ้าของร้าน"

| หัวข้อ | รายละเอียด |
| --- | --- |
| อายุ / อาชีพ | 35 ปี เจ้าของร้านคอมพิวเตอร์ขนาดเล็ก ดูแลทั้งหน้าร้านและออนไลน์คนเดียว |
| ความรู้ด้านเทคนิค | ใช้คอมคล่อง แต่ไม่ใช่โปรแกรมเมอร์ ต้องการหน้าจอที่เข้าใจง่ายไม่ต้องมีความรู้เทคนิค |
| เป้าหมาย | จัดการสต็อกสินค้าและติดตามคำสั่งซื้อทั้งหมดได้จากที่เดียว ไม่ต้องสลับไปมาหลายระบบ รู้ยอดขายภาพรวมได้เร็ว |
| Pain Point | เวลาไม่พอตรวจสอบสลิปโอนเงินทีละออเดอร์ กลัวลืมอัปเดตสถานะให้ลูกค้าจนลูกค้าโทรมาถาม กลัวกดลบสินค้าผิดโดยไม่ได้ตั้งใจ |
| พฤติกรรมการใช้เว็บ | เข้าหน้าแอดมินทุกเช้าเพื่อดูยอดขายและออเดอร์ใหม่ อัปเดตสถานะคำสั่งซื้อหลังเช็คยอดโอนจากแอปธนาคารเสร็จ |
| สถานการณ์ใช้งาน (Scenario) | เข้าสู่ระบบด้วยบัญชีแอดมิน ไปหน้า /admin ดูแดชบอร์ดยอดขายวันนี้ เพิ่มสินค้าใหม่ที่เพิ่งเข้าสต็อก ตรวจตาราง "จัดการคำสั่งซื้อ" เปลี่ยนสถานะออเดอร์ที่ได้รับเงินแล้วเป็น "ชำระเงินแล้ว" ทีละรายการ |
| สิ่งที่ระบบต้องรองรับ | จัดการสินค้า (เพิ่ม/แก้ไข/ลบ) พร้อมกันยืนยันก่อนลบ, ดูแดชบอร์ดสรุปยอดขาย, ดู/เปลี่ยนสถานะคำสั่งซื้อทุกรายการ |

## สรุปความเชื่อมโยงกับฟีเจอร์ในระบบ

| Persona | ฟีเจอร์หลักที่ใช้ | หน้าจอที่เกี่ยวข้อง |
| --- | --- | --- |
| น้องเฟิร์ส (Guest) | ค้นหา/กรองหมวดหมู่, ดูรายละเอียดสินค้า, ตะกร้า | Home, ProductDetail, Cart |
| พี่มายด์ (Customer) | สมัคร/เข้าสู่ระบบ, ยืนยันคำสั่งซื้อ, ติดตามสถานะ | Cart, OrderDetail, History |
| พี่เอก (Admin) | จัดการสินค้า, จัดการคำสั่งซื้อ, แดชบอร์ด | Admin |
